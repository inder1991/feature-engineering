# SP-0 Foundations Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the event-sourced, roll-your-own durable-workflow foundation (event store, immutable document DAG, versioned state machines, atomic outbox/queue runtime, identity + structural SoD, governance/privacy machinery) that every later sub-project (SP-1…SP-12) plugs handlers, gates, commands, and queries into.

**Architecture:** Four aggregates (Request, Feature, Run, immutable Version) each with their own append-only, globally-sequenced event stream are the single source of truth; read models are rebuildable projections checkpointed by `global_seq`. A single atomic Postgres transaction per step appends events + frozen documents + timers + outbox messages, and a leased relay plus a SKIP-LOCKED worker queue drive idempotent step handlers forward across crashes. SP-0 provides the *mechanism* (typed slots, guards, events, aggregates) to represent and enforce governance; the *policies/values* live in SP-9/SP-10/SP-12.

**Tech Stack:** Python 3.11+, PostgreSQL 15+ (event store, transactional outbox, durable timers, worker queue via `SELECT … FOR UPDATE SKIP LOCKED`, projections + checkpoints, processed-message ledger, `external_commands`, `attempt_memory`, `security_audit`, `feature_versions`, `consumers`, documents-index), an S3-compatible object store for blob bodies, OIDC (humans) + workload identity (services), a KMS for per-body encryption + crypto-shred, and pytest for tests.

## Global Constraints

Every task's requirements implicitly include this section. Version floors and the load-bearing invariants are copied verbatim from the SP-0 design spec; they are non-negotiable.

- **Language floor:** Python 3.11+ (`from __future__ import annotations`; native `tuple[...]` / `X | None` typing; `dataclass(frozen=True, slots=True)`).
- **Database floor:** PostgreSQL 15+ (the SKIP-LOCKED worker queue, partial unique indexes, and `MERGE`/upsert mechanics assume 15+).
- **Test floor:** pytest (every task is TDD: write the failing test first, watch it fail, implement minimally, watch it pass, commit).
- **INVARIANT — optimistic concurrency:** `stream_version` — **per aggregate instance** — drives **optimistic concurrency: append only if still at the expected version**; enforced by `UNIQUE (aggregate, aggregate_id, stream_version)`. A failed `stream_version` check rolls back the whole transaction.
- **INVARIANT — monotonic global_seq:** `global_seq` is **monotonic across all streams; projections consume in `global_seq` order**; the contract is **strict monotonicity + no gaps that block ordering**.
- **INVARIANT — events & documents are immutable / write-once:** **Events are never edited/deleted (subject to §9). Corrections are compensating events.** Documents are **write-once**; `branch_role` is immutable per document; promotion is an explicit `PRIMARY_SELECTED` event, **no in-place flip**.
- **INVARIANT — audit IS the event stream:** **Audit *is* the event streams** — there is no separate audit log; every command and outcome is an audited event (denials excepted, see below).
- **INVARIANT — one atomic transaction per step:** **One atomic DB transaction per step** appends domain event(s) + document(s) + timer rows + processed-message ledger + outbox message(s); a failed `stream_version` check rolls back the whole transaction.
- **INVARIANT — handlers idempotent:** A step **handler is idempotent**; duplicate message/outbox-publish yields **one effect** (expected-`stream_version` append + processed-message ledger keyed by message id).
- **INVARIANT — guards pure, may not read mutable projections:** Guard predicates are **pure and deterministic: no side effects, no I/O, and may read frozen documents/version-attributes but not mutable projections** (so replay is deterministic).
- **INVARIANT — denials go to the security stream:** Unauthorized attempts are recorded in the **security-audit stream (append-only, tamper-evident), not the domain stream** — avoiding existence leakage / governed-stream spam.
- **INVARIANT — raw PII never inline:** **No raw PII/secrets in events or document bodies — references only**; sensitive bodies (incl. Draft `raw_input`) live in an encrypted, access-controlled blob store, referenced by id + hash + classification.
- **INTEGRATION — real-migrations suite:** A `tests/integration/` suite MUST run **only the canonical migrations module (`sp0/db/migrations.py`)** — never shadow/prerequisite SQL fixtures — and exercise a **cross-phase happy path**, proving the canonical migrations are sufficient and consistent. Narrow per-unit prerequisite fixtures may remain for fast unit tests, but they must not stand in for the canonical migrations in the integration suite.

## Database schema (shared DDL)

This is the authoritative physical contract. Phase authors create these objects in their owning phase (see the Phase breakdown), but **must not redefine columns, constraints, or names** declared here. All `*_id` text keys are ULID-style prefixed strings (`evt_…`, `doc_…`, `run_id` = `run_…`, etc.).

### Global sequence (monotonic `global_seq`)

```sql
-- Single logical allocator for global_seq across ALL streams (events, documents,
-- security_audit). Strict monotonicity is the contract; gaps that do not block
-- ordering are acceptable. Cache 1 to keep gaps minimal under concurrency.
CREATE SEQUENCE global_seq_seq AS bigint INCREMENT BY 1 START WITH 1 NO CYCLE CACHE 1;
```

-- =========================================================================
-- events — the source of truth (§3.2). One row per domain event, all streams.
-- =========================================================================
CREATE TABLE events (
    event_id        text        PRIMARY KEY,                      -- 'evt_...'
    global_seq      bigint      NOT NULL DEFAULT nextval('global_seq_seq'),
    aggregate       text        NOT NULL CHECK (aggregate IN ('request','feature','run')),
    aggregate_id    text        NOT NULL,                         -- request_id | feature_id | run_id
    stream_version  integer     NOT NULL CHECK (stream_version > 0),
    request_id      text        NULL,                             -- null on pure feature-stream events
    feature_id      text        NULL,                             -- null on pure request-stream events
    run_id          text        NULL,                             -- null on request-/feature-stream events
    type            text        NOT NULL,                         -- registered event type
    schema_version  integer     NOT NULL,                         -- stamped at write (§3.3)
    table_version   integer     NOT NULL,                         -- pinned state-machine table version (§4.2)
    actor           jsonb       NOT NULL,                         -- IdentityEnvelope (§6.1)
    payload         jsonb       NOT NULL,
    provenance      jsonb       NOT NULL,                         -- ProvenanceEnvelope (§8)
    caused_by       text        NULL REFERENCES events(event_id),
    occurred_at     timestamptz NOT NULL,
    recorded_at     timestamptz NOT NULL DEFAULT now(),
    -- optimistic concurrency: append only if still at expected stream_version
    CONSTRAINT events_optimistic_concurrency UNIQUE (aggregate, aggregate_id, stream_version),
    CONSTRAINT events_global_seq_unique       UNIQUE (global_seq),
    -- aggregate_id must match the column for its aggregate kind
    CONSTRAINT events_aggregate_id_consistent CHECK (
        (aggregate = 'request' AND aggregate_id = request_id) OR
        (aggregate = 'feature' AND aggregate_id = feature_id) OR
        (aggregate = 'run'     AND aggregate_id = run_id)
    )
);
CREATE INDEX events_stream_idx   ON events (aggregate, aggregate_id, stream_version);
CREATE INDEX events_global_idx   ON events (global_seq);

CREATE TABLE documents (

    content_hash        text        NOT NULL,                     -- 'sha256:...'

    actor               jsonb       NOT NULL,
    provenance          jsonb       NOT NULL,

        'FEATURE_PLAN','CANDIDATE_SQL','VALIDATION_REPORT','SANDBOX_RESULT','DQ_REPORT',

CREATE TABLE queue (

-- Physical immutability (no UPDATE/DELETE) — mirrors the documents write-once trigger.
-- The `immutable` boolean flag/convention is NOT sufficient; this is the backstop.
CREATE OR REPLACE FUNCTION prevent_feature_version_mutation() RETURNS trigger AS $$
BEGIN
    RAISE EXCEPTION 'feature_versions are immutable: % not allowed on feature_version_id=%',
        TG_OP, COALESCE(OLD.feature_version_id, NEW.feature_version_id);
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER feature_versions_no_mutation
    BEFORE UPDATE OR DELETE ON feature_versions
    FOR EACH ROW EXECUTE FUNCTION prevent_feature_version_mutation();

    activated_seq      bigint      NOT NULL,                      -- global_seq of the activation event

);

-- =========================================================================

    registered_by      jsonb       NOT NULL,                      -- IdentityEnvelope

    UNIQUE (feature_id, consumer_kind, consumer_ref)

CREATE INDEX consumers_feature_active_idx ON consumers (feature_id) WHERE edge_status = 'active';

    guard_expr    text        NULL,

);
CREATE TABLE feature_lifecycle_table (LIKE run_transition_table INCLUDING ALL);

    `degraded` (e.g. run_workflow_state.degraded=true) and stop advancing it; analytics
    projections may log-and-skip."""

## Notes for phase authors

- **Do not redefine shared symbols.** If a signature here is wrong or insufficient, raise it back to this overview rather than diverging — a `clearLayers()`-vs-`clearFullLayers()` mismatch across phases is a build-breaking bug.
- **Every step is TDD** (pytest): failing test → watch it fail → minimal implementation → watch it pass → commit. Use the Global Constraints invariants as test oracles (e.g. a second append at a stale `expected_version` MUST raise `ConcurrencyError`; a denied command MUST land in `security_audit` and NOT in `events`).
- **The §5.1 atomic boundary is one transaction**: append event(s) (OCC check) + insert document(s) + upsert timer rows + record processed-message ledger + insert outbox message(s), COMMIT. Any `ConcurrencyError` rolls back the whole transaction.
- **Phase ownership of shared DDL:** the table is declared once here; the owning phase creates it (migration) and the consuming phases reference it. Cross-phase references (e.g. Phase 05's blob GC over Phase 02's `blob_index`, Phase 07's authz hook into Phase 06's `execute_command`) are intentional and listed above.
- **Error-handling/test coverage** required by spec §12 is allocated to the phase that owns the mechanism under test; each phase plan must include those cases explicitly (no "write tests for the above").
