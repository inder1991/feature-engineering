def _is_degraded(conn: DbConn, cmd: Command) -> bool:

### Task 2: Event-sourced resolve + rebuild clears degraded

**Findings B2 + rebuild-degraded.** `resolve_degraded_command` clears `run_workflow_state.degraded` with no domain event (and now that Task 1 reads `projection_degraded`, it clears the wrong table); `rebuild_projection` clears `projection_skips` but not `projection_degraded` (stale rows become false blockers under Task 1).

**Design decision (why this shape):** `projection_degraded` is a **runner-derived operational ledger** — on rebuild the runner re-marks it if the poison persists, and does not if the poison was remediated. So resolution does **not** need to be *replayed* to clear it; the clear is implicit in "poison fixed → runner doesn't re-mark." `resolve_degraded_command` therefore (a) **deletes** the `projection_degraded` row(s) for the aggregate (direct write to a re-derivable operational table — allowed), and (b) **appends a `DEGRADED_RESOLVED` domain event** as the immutable audit record of the operator's remediation decision (who/when/why). `rebuild_projection` clears `projection_degraded` before replay so it is re-derived cleanly.

**Files:** Modify `src/featuregen/aggregates/run_lifecycle.py` (`resolve_degraded_command`, ~line 172), the run event-schema registration (`aggregates/events.py`), `src/featuregen/projections/runner.py` (`rebuild_projection`, ~line 128). Tests: `tests/featuregen/aggregates/test_run_lifecycle.py`, `tests/projections/test_rebuild.py`.

**Interfaces:**
- Produces: event type `DEGRADED_RESOLVED` (payload `{run_id, aggregate, aggregate_id, reason?}`); `resolve_degraded_command` deletes `projection_degraded` for the aggregate + appends `DEGRADED_RESOLVED`; `rebuild_projection` clears `projection_degraded` for the projection before replay.

```python
def resolve_degraded_command(conn: DbConn, cmd: Command) -> CommandResult:
    """Clear a degraded aggregate after remediation (§3.6). projection_degraded is a
    runner-derived operational ledger (re-marked on rebuild if the poison persists), so we
    DELETE it directly AND append DEGRADED_RESOLVED as the immutable audit of the remediation
    (review B2). execute_command special-cases this action so it is not self-blocked."""
    aggregate = cmd.args.get("aggregate", "run")
    aggregate_id = cmd.aggregate_id
    conn.execute(
        "DELETE FROM projection_degraded WHERE aggregate=%s AND aggregate_id=%s",
        (aggregate, aggregate_id),
    )
    evt = append(
        conn, aggregate="run", aggregate_id=aggregate_id, type="DEGRADED_RESOLVED",
        payload={"run_id": aggregate_id, "aggregate": aggregate, "aggregate_id": aggregate_id,
                 "reason": cmd.args.get("reason")},
        actor=cmd.actor, run_id=aggregate_id,
    )
    return CommandResult(accepted=True, aggregate_id=aggregate_id or "",
                         produced_event_ids=(evt.event_id,))
```
