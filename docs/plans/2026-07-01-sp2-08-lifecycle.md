This phase closes the SP-2 loop. It delivers the **contract lifecycle as a fold + inline-guard state
machine** over SP-2's own **`feature_contract` event stream** — **`fold_feature_contract_state(stream) ->
FeatureContractState`** (`intake/state.py`, mirroring SP-1's `overlay/state.py::fold_overlay_state`) plus the
pure fold-companion guards each command handler evaluates **before** appending (mirroring
`overlay/confirmation_commands.py` gating on the folded `status`, **incl. the no-regression guard**). It is
**explicitly NOT** SP-0's declarative `state_machine/engine.py` (built-but-unused — SP-1 deliberately did not
wire it), **NOT** the `run_workflow_state` projection (unwired scaffold), and **NOT** a document-`Stage`
transition (that enum is artifact taxonomy, spec §4.6). The closed status vocabulary is
`NEEDS_CLARIFICATION → MINIMUM_CONTRACT_VALIDATED → CONFIRMED`, plus the branch/terminal statuses
`OUT_OF_SCOPE` / `PROHIBITED_DATA_CLASS` / `NEEDS_USE_CASE_ONBOARDING` (spec §4.6, §11).

On top of the fold it wires the two **non-confirming exits** (spec §11, §13, Decision D13):

- **`reject_intent`** — the **platform/service-issued** deterministic-classifier terminal outcome for
  `OUT_OF_SCOPE` / `PROHIBITED_DATA_CLASS`, appending `INTENT_REJECTED` on the **`feature_contract`** aggregate
  (folding the status to the matching terminal) **and** driving SP-0's **`RUN_REJECTED`** on the **run**
  aggregate. It runs as `service:intake-agent` under the **additive** `("reject_intent","","intake-agent",
  "service",None)` authz row (§2.1 #5) — SP-2 **never** invokes SP-0's validator-only `reject`
  (`authz/policy.py:42`).
- **requester `withdraw` reuse** — requester-initiated abandonment (e.g. a Gate #1 `reject` response) reuses
  **SP-0's data-scientist-owned `withdraw`** (`authz/policy.py:41`, → `RUN_WITHDRAWN`) behind SP-2's
  request-owner guard; it adds **no** authz row and emits **no** `feature_contract` event (withdrawal is a
  run-level terminal, surfaced by `run_is_terminal`).

Finally it builds the **read interface SP-3 binds to** — `get_contract(conn, run_id) -> ContractView` (§13,
service-internal, **fail-closed** exactly like SP-1's `resolve_fact`: a usable Confirmed contract is served
**only** when the folded status is `CONFIRMED` and the run is not terminal) — and, **optionally/secondary to
the fold**, a fail-closed status-projection **query** helper (`read_contract_status`, never consulted for a
command decision; a stored `OverlayProjection`-style table is out of scope — SP-2 ships exactly two migrations,
both in P1).

The `feature_contract` aggregate is keyed by **`aggregate_id == run_id`** (one contract per run), admitted by
P1's `0508` event-store aggregate-CHECK widening (the typed `feature_contract_id` mirror, mirroring SP-1's
`0504` `overlay_fact_id`). The fold loads its stream with `load_stream(conn, "feature_contract", run_id)`; every
`feature_contract` append sets `feature_contract_id=run_id` and leaves the `run_id`/`request_id`/`feature_id`
mirror columns NULL (the id-consistency branch), so run-terminal outcomes (`RUN_REJECTED`/`RUN_WITHDRAWN`) live
on the **run** aggregate and correlate by the shared id string.

**Depends on:** P1 (the `0508`/`0509` migrations; the twelve FC event-type schemas + `register_sp2_event_types`;
`append(...)` extended with the `feature_contract_id` mirror keyword; the `reject_intent` service authz row) ·
P2–P7 (the FC events these handlers emit; the P5 `mcv.py` checklist; the P7 Gate #1 confirm path). All tests run
on the `db` fixture with `FakeLLM`-free, event-level fixtures.

```python
"""The `feature_contract` aggregate status FOLD (SP-2 design §4.6, §11).

SP-2's Feature Contract lifecycle is its OWN event-sourced aggregate; its status is FOLDED from the
event stream — never a stored enum, never a projection row — mirroring SP-1's
`overlay/state.py::fold_overlay_state`. This is the authoritative state every command handler decides
against (inline, before appending). It is NOT `state_machine/engine.py` and NOT `run_workflow_state`.

The aggregate is keyed by `aggregate_id == run_id` (one contract per run). `append_fc_event` sets the
`feature_contract_id` mirror (P1's `0508` id-consistency branch) and leaves the run/request/feature
mirror columns NULL; run-terminal outcomes (RUN_REJECTED/RUN_WITHDRAWN) live on the `run` aggregate.
"""

from __future__ import annotations

from collections.abc import Iterable, Mapping
from dataclasses import dataclass
from enum import Enum
from typing import Any

import featuregen.intake.events as ev
from featuregen.aggregates._append import append
from featuregen.contracts import DbConn, EventEnvelope, IdentityEnvelope
from featuregen.events.store import load_stream


class FeatureContractStatus(str, Enum):
    """The closed lifecycle vocabulary (§4.6). Extended only by a schema bump + upcaster."""

    NEEDS_CLARIFICATION = "NEEDS_CLARIFICATION"
    MINIMUM_CONTRACT_VALIDATED = "MINIMUM_CONTRACT_VALIDATED"
    CONFIRMED = "CONFIRMED"
    OUT_OF_SCOPE = "OUT_OF_SCOPE"                     # terminal (banking-boundary)
    PROHIBITED_DATA_CLASS = "PROHIBITED_DATA_CLASS"   # terminal (blocked-class)
    NEEDS_USE_CASE_ONBOARDING = "NEEDS_USE_CASE_ONBOARDING"  # park / hold


# Sticky terminal set: once folded here, later status-MUTATING events are IGNORED (no regression) —
# the analogue of fold_overlay_state's defensive "stray PROPOSED after a confirm — ignore" branch.
_TERMINAL_STATUSES: tuple[FeatureContractStatus, ...] = (
    FeatureContractStatus.CONFIRMED,
    FeatureContractStatus.OUT_OF_SCOPE,
    FeatureContractStatus.PROHIBITED_DATA_CLASS,
)
# After a terminal fold, only pure-audit events may still be folded (they never touch status/content).
_POST_TERMINAL_ALLOWED: frozenset[str] = frozenset({ev.LLM_CALL_RECORDED})


@dataclass(frozen=True, slots=True)
class FeatureContractState:
    status: FeatureContractStatus | None
    open_fields: tuple[str, ...]
    requester: str | None
    request_id: str | None
    intake_mode: str | None
    catalog_version: str | None
    draft_doc_id: str | None
    confirmed_doc_id: str | None
    assumption_ledger_ref: str | None
    proposed_feature_name: str | None
    feature_name: str | None
    field_scores: Mapping[str, Any] | None
    open_questions: tuple[Any, ...]
    requires_independent_validation: bool | None
    selected_candidate: str | None
    rejection_classification: str | None
    rejection_reason: str | None
    matched_class: str | None
    llm_call_refs: tuple[str, ...]


def fold_feature_contract_state(stream: Iterable) -> FeatureContractState:
    """Fold a `feature_contract` event stream (stream_version ASC) into the current lifecycle state
    (§4.6, §11). Mirrors `fold_overlay_state`: each item exposes `.type` and `.payload`. Pure and
    deterministic. NO-REGRESSION: once the fold reaches a terminal status, any later status/content
    event is ignored (only pure-audit LLM_CALL_RECORDED still folds)."""
    status: FeatureContractStatus | None = None
    open_fields: tuple[str, ...] = ()
    acc: dict[str, Any] = {
        "requester": None, "request_id": None, "intake_mode": None, "catalog_version": None,
        "draft_doc_id": None, "confirmed_doc_id": None, "assumption_ledger_ref": None,
        "proposed_feature_name": None, "feature_name": None, "field_scores": None,
        "open_questions": (), "requires_independent_validation": None, "selected_candidate": None,
        "rejection_classification": None, "rejection_reason": None, "matched_class": None,
        "llm_call_refs": (),
    }
    for event in stream:
        t = event.type
        p = event.payload
        if status in _TERMINAL_STATUSES and t not in _POST_TERMINAL_ALLOWED:
            continue  # no-regression: a terminal/confirmed fold refuses a conflicting re-advance
        if t == ev.INTENT_SUBMITTED:
            status = FeatureContractStatus.NEEDS_CLARIFICATION
            acc["requester"] = p.get("requester") or p.get("submitted_by")
            acc["request_id"] = p.get("request_id")
            acc["intake_mode"] = p.get("intake_mode")
            acc["catalog_version"] = p.get("catalog_version")
        elif t == ev.DRAFT_CONTRACT_PRODUCED:
            acc["draft_doc_id"] = p.get("draft_doc_id")
            acc["assumption_ledger_ref"] = p.get("assumption_ledger_ref")
            acc["proposed_feature_name"] = p.get("proposed_feature_name")
            if p.get("field_scores") is not None:
                acc["field_scores"] = p["field_scores"]
            acc["open_questions"] = tuple(p.get("open_questions", ()))
            open_fields = tuple(p.get("open_fields", ()))
        elif t == ev.FIELD_AUTO_RESOLVED:
            field = p.get("field")
            open_fields = tuple(f for f in open_fields if f != field)
        elif t == ev.CONTRACT_REFINED:
            if p.get("draft_doc_id"):
                acc["draft_doc_id"] = p["draft_doc_id"]
            if "open_fields" in p:
                open_fields = tuple(p["open_fields"])
            if p.get("field_scores") is not None:
                acc["field_scores"] = p["field_scores"]
            if p.get("open_questions") is not None:
                acc["open_questions"] = tuple(p["open_questions"])
            # An edit that RE-OPENS a field regresses a validated contract back to clarification.
            if status is FeatureContractStatus.MINIMUM_CONTRACT_VALIDATED and open_fields:
                status = FeatureContractStatus.NEEDS_CLARIFICATION
        elif t == ev.MINIMUM_CONTRACT_VALIDATED:
            status = FeatureContractStatus.MINIMUM_CONTRACT_VALIDATED
        elif t == ev.CONTRACT_CONFIRMED:
            status = FeatureContractStatus.CONFIRMED
            acc["confirmed_doc_id"] = p.get("confirmed_doc_id")
            acc["feature_name"] = p.get("feature_name") or acc["proposed_feature_name"]
            acc["requires_independent_validation"] = p.get("requires_independent_validation")
            acc["selected_candidate"] = p.get("selected_candidate")
        elif t == ev.USE_CASE_ONBOARDING_REQUESTED:
            status = FeatureContractStatus.NEEDS_USE_CASE_ONBOARDING
            acc["catalog_version"] = p.get("catalog_version", acc["catalog_version"])
        elif t == ev.INTENT_REJECTED:
            classification = p.get("classification")
            status = (
                FeatureContractStatus.OUT_OF_SCOPE
                if classification == "OUT_OF_SCOPE"
                else FeatureContractStatus.PROHIBITED_DATA_CLASS
            )
            acc["rejection_classification"] = classification
            acc["rejection_reason"] = p.get("reason")
            acc["matched_class"] = p.get("matched_class")
            acc["catalog_version"] = p.get("catalog_version", acc["catalog_version"])
        elif t == ev.LLM_CALL_RECORDED:
            ref = p.get("llm_call_ref")
            if ref:
                acc["llm_call_refs"] = acc["llm_call_refs"] + (ref,)
        # CONTRACT_CRITIQUED / CLARIFICATION_REQUESTED / CLARIFICATION_ANSWERED: no status change.
    return FeatureContractState(status=status, open_fields=open_fields, **acc)


def load_contract_stream(conn: DbConn, run_id: str) -> list[EventEnvelope]:
    """Load the `feature_contract` aggregate stream for a run (aggregate_id == run_id)."""
    return load_stream(conn, "feature_contract", run_id)


def append_fc_event(
    conn: DbConn,
    *,
    run_id: str,
    type: str,
    payload: Mapping[str, Any],
    actor: IdentityEnvelope,
    caused_by: str | None = None,
    expected_version: int | None = None,
) -> EventEnvelope:
    """Append one `feature_contract` event (aggregate_id == feature_contract_id == run_id). Mirror of
    SP-1's `append_overlay_event`; sets P1's `feature_contract_id` mirror and leaves the run/request/
    feature mirror columns NULL (the `0508` id-consistency branch)."""
    return append(
        conn,
        aggregate="feature_contract",
        aggregate_id=run_id,
        feature_contract_id=run_id,
        type=type,
        payload=dict(payload),
        actor=actor,
        caused_by=caused_by,
        expected_version=expected_version,
    )
```

```python
def is_terminal(state: FeatureContractState) -> bool:
    """True once the fold is at a terminal status (CONFIRMED / OUT_OF_SCOPE / PROHIBITED_DATA_CLASS)."""
    return state.status in _TERMINAL_STATUSES


def open_fields_empty(state: FeatureContractState) -> bool:
    """The Gate-#1 hard invariant (§11): a run with a non-empty open_fields can never advance."""
    return len(state.open_fields) == 0


def actor_is_request_owner(state: FeatureContractState, actor: IdentityEnvelope) -> bool:
    """The SP-2-built request-owner guard SP-0 does NOT provide (§2.1, §6.5, §8.2): SP-0 admits the
    `data_scientist` role for ANY human, so role-authz alone would let any data scientist act on
    another author's contract. The request owner is folded from INTENT_SUBMITTED (`requester`)."""
    return state.requester is not None and actor.subject == state.requester


def confirmer_is_requester_human(state: FeatureContractState, actor: IdentityEnvelope) -> bool:
    """Gate #1 / withdrawal guard (§8.2): the acting subject is the request owner AND a human — a
    service (or the LLM) can never confirm or abandon a contract."""
    return actor.actor_kind == "human" and actor_is_request_owner(state, actor)


def guard_advance(
    state: FeatureContractState,
    allowed_from: tuple[FeatureContractStatus, ...],
) -> str | None:
    """The inline lifecycle guard every advancing handler runs BEFORE appending (§11), mirroring
    `overlay/confirmation_commands.py`. Returns a deny reason, or None to proceed. Enforces (a) an
    opened contract, (b) NO-REGRESSION (a terminal/confirmed fold refuses a conflicting re-advance),
    and (c) the allowed source status."""
    if state.status is None:
        return "no feature contract for this run"
    if is_terminal(state):
        return f"contract already terminal (status={state.status.value})"
    if state.status not in allowed_from:
        return f"illegal advance from {state.status.value}"
    return None
```

```python
`reject_intent` — the platform/service-issued deterministic-classifier terminal outcome for
`OUT_OF_SCOPE`/`PROHIBITED_DATA_CLASS` (INTENT_REJECTED on the feature_contract aggregate + SP-0
RUN_REJECTED on the run). `withdraw_intent` — requester-initiated abandonment reusing SP-0's
data-scientist-owned `withdraw` (RUN_WITHDRAWN) behind SP-2's request-owner guard. SP-2 NEVER invokes
SP-0's validator-only `reject` action (`authz/policy.py:42`), and withdrawal adds no authz row.
"""

from __future__ import annotations

from dataclasses import replace

import featuregen.intake.events as ev
from featuregen.aggregates.run_lifecycle import reject_command, withdraw_command
from featuregen.contracts import Command, CommandResult, DbConn
from featuregen.intake.state import (
    FeatureContractStatus,
    actor_is_request_owner,
    append_fc_event,
    fold_feature_contract_state,
    guard_advance,
    is_terminal,
    load_contract_stream,
)
from featuregen.security.audit import record_denial

# A banking-boundary rejection may be issued from either the initial classification (before the
# Draft is even scored) or after MCV (the §8.4 authoritative re-screen at Gate #1).
_REJECTABLE_FROM: tuple[FeatureContractStatus, ...] = (
    FeatureContractStatus.NEEDS_CLARIFICATION,
    FeatureContractStatus.MINIMUM_CONTRACT_VALIDATED,
)
_REJECT_CLASSIFICATIONS = ("OUT_OF_SCOPE", "PROHIBITED_DATA_CLASS")


def _deny_audited(conn: DbConn, cmd: Command, run_id: str, reason: str) -> CommandResult:
    """Route an authority/owner denial to the tamper-evident security-audit stream (§6.2), then deny —
    mirroring `overlay/_lifecycle.py::_deny_audited` (the coarse PolicyAuthorizer only audits role/kind,
    so a fine-grained handler denial would otherwise leave no trace)."""
    record_denial(conn, replace(cmd, aggregate_id=run_id), reason)
    return CommandResult(accepted=False, aggregate_id=run_id, denied_reason=reason)


def reject_intent(conn: DbConn, cmd: Command) -> CommandResult:
    """Platform/service intake rejection (§5.4, §8.4, §11). Folds the FC status, runs the inline
    no-regression guard, appends INTENT_REJECTED (OCC-pinned) so the fold advances to the matching
    terminal, then drives SP-0's RUN_REJECTED. Idempotent by no-regression: a re-reject of an
    already-terminal contract is denied (never double-charged)."""
    args = cmd.args
    run_id = args["run_id"]
    classification = args.get("classification")
    if classification not in _REJECT_CLASSIFICATIONS:
        return CommandResult(
            accepted=False, aggregate_id=run_id,
            denied_reason=f"reject_intent classification must be one of {_REJECT_CLASSIFICATIONS}, "
                          f"got {classification!r}",
        )
    stream = load_contract_stream(conn, run_id)
    state = fold_feature_contract_state(stream)
    deny = guard_advance(state, _REJECTABLE_FROM)
    if deny is not None:
        return CommandResult(accepted=False, aggregate_id=run_id, denied_reason=deny)

    payload = {"run_id": run_id, "classification": classification,
               "catalog_version": args["catalog_version"]}
    if args.get("reason") is not None:
        payload["reason"] = args["reason"]
    if args.get("matched_class") is not None:
        payload["matched_class"] = args["matched_class"]
    rejected = append_fc_event(
        conn, run_id=run_id, type=ev.INTENT_REJECTED, payload=payload, actor=cmd.actor,
        expected_version=stream[-1].stream_version,  # OCC pin to the folded head (§12)
    )
    # Drive SP-0's run terminal via the existing lifecycle handler (NOT the validator-only `reject`
    # action): `reject_command` reads cmd.aggregate_id (run_id) + cmd.args["reason"] and checks run
    # terminality itself.
    run_cmd = replace(cmd, aggregate="run", aggregate_id=run_id, args={"reason": args.get("reason")})
    run_res = reject_command(conn, run_cmd)
    if not run_res.accepted:
        return run_res
    return CommandResult(
        accepted=True, aggregate_id=run_id,
        produced_event_ids=(rejected.event_id, *run_res.produced_event_ids),
    )
```

```python
def withdraw_intent(conn: DbConn, cmd: Command) -> CommandResult:
    """Requester-initiated abandonment (§11, §13, Decision D13). Reuses SP-0's data-scientist-owned
    `withdraw` (→ RUN_WITHDRAWN) behind SP-2's request-owner guard — NOT the validator-only `reject`.
    Human + data_scientist + request-owner only; appends NO feature_contract event (withdrawal is a
    run-level terminal). Adds no authz row: it delegates to SP-0's existing `withdraw` capability."""
    args = cmd.args
    run_id = args["run_id"]
    if cmd.actor.actor_kind != "human":
        return _deny_audited(conn, cmd, run_id, "withdraw requires a human requester")
    if "data_scientist" not in cmd.actor.role_claims:
        return _deny_audited(conn, cmd, run_id, "withdraw requires the data_scientist role")
    stream = load_contract_stream(conn, run_id)
    if not stream:
        return CommandResult(accepted=False, aggregate_id=run_id,
                             denied_reason="no feature contract for this run")
    state = fold_feature_contract_state(stream)
    if is_terminal(state):
        return CommandResult(accepted=False, aggregate_id=run_id,
                             denied_reason=f"contract already terminal (status={state.status.value})")
    if not actor_is_request_owner(state, cmd.actor):
        return _deny_audited(conn, cmd, run_id,
                             "actor is not the request owner; withdrawal is requester-initiated")
    run_cmd = replace(
        cmd, aggregate="run", aggregate_id=run_id,
        args={"reason": args.get("reason", "requester withdrew intent")},
    )
    return withdraw_command(conn, run_cmd)
```

```python
"""The service-internal read interface SP-3+ binds to (§13) — the SP-2 analogue of SP-1's
`resolve_fact`. FAIL-CLOSED: a Confirmed contract is servable only when the FOLDED status is CONFIRMED
and the run is not terminal; every other status (or a rejected/withdrawn run) blocks with a reason.
The fold is authoritative — this reader never consults a projection for a decision (§11, §12)."""

from __future__ import annotations

from collections.abc import Mapping
from dataclasses import dataclass
from typing import Any

from featuregen.contracts import DbConn
from featuregen.contracts.documents import Stage
from featuregen.documents.store import get_document
from featuregen.events.store import load_stream
from featuregen.intake.state import (
    FeatureContractStatus,
    fold_feature_contract_state,
    load_contract_stream,
)

_RUN_TERMINALS = ("RUN_REJECTED", "RUN_WITHDRAWN", "RUN_CANCELLED")


@dataclass(frozen=True, slots=True)
class ContractView:
    run_id: str
    stage: str
    status: str
    intake_mode: str | None
    feature_name: str | None
    draft_doc_id: str | None
    confirmed_doc_id: str | None
    assumption_ledger_ref: str | None
    field_scores: Mapping[str, Any] | None
    open_questions: tuple[Any, ...]
    open_fields: tuple[str, ...]
    requires_independent_validation: bool | None
    catalog_version: str | None
    selected_candidate: str | None
    terminal_outcome: str | None
    body_ref: str | None
    content_hash: str | None
    reason_if_unavailable: str | None


def _run_terminal_outcome(conn: DbConn, run_id: str) -> str | None:
    for e in load_stream(conn, "run", run_id):
        if e.type in _RUN_TERMINALS:
            return e.type
    return None


def get_contract(conn: DbConn, run_id: str) -> ContractView | None:
    """Fetch the folded Feature Contract view for a run, or None if no `feature_contract` was opened.
    Servable (reason_if_unavailable is None) ONLY for a CONFIRMED, non-terminal-run contract."""
    stream = load_contract_stream(conn, run_id)
    if not stream:
        return None
    state = fold_feature_contract_state(stream)
    terminal_outcome = _run_terminal_outcome(conn, run_id)
    is_confirmed = (
        state.status is FeatureContractStatus.CONFIRMED and terminal_outcome is None
    )
    stage = Stage.CONFIRMED_CONTRACT.value if is_confirmed else Stage.DRAFT_CONTRACT.value
    doc_id = state.confirmed_doc_id if is_confirmed else state.draft_doc_id

    body_ref = content_hash = None
    if doc_id is not None:
        doc = get_document(conn, doc_id)
        if doc is not None:
            body_ref = doc.get("body_ref")
            content_hash = doc.get("content_hash")

    if terminal_outcome is not None:
        reason: str | None = f"run terminal: {terminal_outcome}"
    elif state.status is not FeatureContractStatus.CONFIRMED:
        status_label = state.status.value if state.status is not None else "MISSING"
        reason = f"not confirmed (status={status_label})"
    else:
        reason = None

    return ContractView(
        run_id=run_id,
        stage=stage,
        status=state.status.value if state.status is not None else "MISSING",
        intake_mode=state.intake_mode,
        feature_name=state.feature_name or state.proposed_feature_name,
        draft_doc_id=state.draft_doc_id,
        confirmed_doc_id=state.confirmed_doc_id,
        assumption_ledger_ref=state.assumption_ledger_ref,
        field_scores=state.field_scores,
        open_questions=state.open_questions,
        open_fields=state.open_fields,
        requires_independent_validation=state.requires_independent_validation,
        catalog_version=state.catalog_version,
        selected_candidate=state.selected_candidate,
        terminal_outcome=terminal_outcome,
        body_ref=body_ref,
        content_hash=content_hash,
        reason_if_unavailable=reason,
    )
```
